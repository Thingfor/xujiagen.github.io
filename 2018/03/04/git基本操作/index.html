git基本操作

发表于 2018-03-04 | 分类于 git

在git里面链接远程仓库的方法有两个：

1.在git bash里面：

命令名称 解释 作用
mkdir 文件夹名称 在本地创建一个文件夹
cd 文件夹名称 跳转到刚刚创建的文件夹里面
git init 跳转到刚才的文件夹之后，将刚才的文件夹标记为本地默认的仓库
touch index.html 创建一个文件
echo “hello world”>index.html 创建的index.html里面通过echo对文件里面写入内容
git add . 将本地的文件上传到缓存，.号表示将刚才的所有文件都上传到缓存里面
git commit -m ‘对更新的描述’ 对刚才上传的文件缓存描述
git push origin master 将刚才在缓存里面的文件上传到云端仓库里面，通过master分支

2.登录到github里面，创建一个仓库，将这个仓库的地址记录下来
在本地的git bash里面执行：

git clone 仓库地址

注意：–global 表示对于git里面所有的仓库都是用这两个设置
命令名称|解释|作用
—-|—-|—-
git config –global user.name “xujiagen”| |这里config表示系统设置
git config –global user.email “xjiagen@gmail.com“| |这里的设置使用者的邮箱
mkdir learsngit|创建一个文件夹
cd learngit|跳转到创建的文件夹
pwd|显示当前位置的文件夹目录
git init| |git将当前目录设置为一个仓库，应该是本地的，push之后远程也会创建一个仓库

假如我们在刚才的文件夹里面创建一个txt文件，并且，文件里面已经写了一定的内容

这个时候我们通过:

git add 文件名.txt //将文件放入到仓库之中
git commit -m “我创建的一个新的文件” //这个时候我们备注上一些对这个文件的描述
//其中-m表示本次描述的内容

注意：git add 文件名1.txt 文件名2.txt 都是可以的
git add . //表示向仓库里面加入当前目录下所有的文件

如果如果我们刚才创建的文件被修改了
git status //可以看到我们当前仓库的状态，那些文件被修改了

git diff 文件名.txt //表示刚才修改的文件具体修改的内容

在github里面有两个区，三种环境
两个区：工作区，版本库
三种环境是：工作区，暂存区，HEAD区
工作区->暂存区->HEAD区
工作区(本地)，add之后文件存放在暂存区里面，push之后文件正式存放在HEAD区里面

命令名称 解释 作用
git check –文件名 是将还没有上传到HEAD区里面的数据撤销掉

这里分成两种情况：

1.文件没有提交到暂存区里面去了，这里是撤销修改
2.文件已经提交到暂存区里面去了，这里是撤销到修改之前

命令名称 解释 作用
git log 显示在github上面修改的次数，以及每次修改的内容
git log –pretty=oneline 单独显示的修改次数可能很多，可以将显示的内容都存储在一行上面，这样比较简洁

git reset –hard HEAD^ //表示将文件会退到上次修改的时候，这里只是将准备提交的文件回退，但是你如果想要放弃本地的修改你需要：git checkout –文件名 将在本地文件中修改的内容会退到之前没有修改的程度

当然如果修改的次数太多了，想要会退到特定的版本可以使用：
git reset –hard （commit ID） //将文件会退到commit ID特定的ID的时期，这个ID不需要完整，只需要与其他的ID不一样，便于区别就可以了

当然你回退之后，还可以再次会退到回退之前：
git reflog //显示你可以回退的ID，按照刚才打的ID就可以修改成为你想要的版本时期

git diff HEAD –文件名 可以看看工作区里面和版本库里面的区别

当然你可以修改在版本库里面存储好的文件
git rm 文件名

git checkout //是本地文件在修改之后，我发现修改不好，将云端的所有文件同步到本地
git checkout –文件名 //可以将特定的文件从云端同步到本地，替换掉本地的修改

云端的版本库与本地的工作区之间通信是ssh形式，这个时候你需要在本地创建秘钥，再将秘钥复制到远程的github账户里面去，这样，你就可以让本地的工作区与云端的仓库自由的通信了。
ssh-keygen -t rsa -C “邮箱名称”获取你本地创建的秘钥，然后粘贴到云端。

git remote add origin git@github.com:账户名/仓库名.git //将远程的仓库克隆到本地
git push -u origin master //修改之后，将本地的文件推送到云端

git checkout -b 分支名称 //创建一个新的分支，并且切换到新的分支上面

git branch 分支名称 //创建一个新的分支
git checkout 分支名称 //切换到刚才创建的分支上面

git branch //查看现有的所有分支
git merge 其它分支 //合并其他分支到当前分支上面来
git branch -d 分支名称 //删除某分支，注注意删除的分支不应该是工作分支

当两个分支都有文件往上面上传的时候需要注意注意，这个时候是不可以将两个分支进行合并的
git merge –no-ff -m “提交” 分支名称

git stash //将修改临时保存在一个地方，但是当前的工作区恢复到之前没有修改的样子

删除文件

git rm 文件名 将文件删除
git checkout –文件名 将本地被删除，云端文件还存在的文件恢复
