生成器

生成器与列表生成器的区别在于，生成器生成的数据可以非常多，但是不会占用大量的内存，因为他是实时计算出来的，使用的时候才开始调用。

生成器的使用方式不仅仅是像列表生成器那样通过一个[]，对于一个函数如果将函数里面的数据通过yield操作，这个函数就相当于一个生成器

def fib(max):
n,a,b=0,0,1
while n<max:
yield b
a,b=b,a+b
n+=1

list,tuple,dict,set,str

函数本身也可以赋值给一个变量，变量可以指向函数
当一个变量指向一个函数的时候，可以通过这个变量调用这个函数
那么函数名就是指向函数的变量

既然变量可以指向函数，函数的参数能接受变量，那么一个函数可以接收另一个函数作为参数，这种函数被称为高阶函数

def(x,y,f):

filter函数用于过滤序列

filter函数接收一个函数和一个序列，filter把传入的函数作用于每一个元素，按照返回的阿寒湖是true还是false判断是否丢弃这个元素

就相当于在c++里面的谓词，通过这个谓词判断是否保留元素

高阶函数排序函数，排序函数可以接收三个元素，第一个是一个list/tuple,第一个是一个处理函数，第三个是reverse=True
正常情况下函数都是从小到大进行排序的

k=[12,-34,31,432,-223,122]

函数的返回结果是一个函数，然后运行函数就可以得到刚才的结果了

返回闭包的时候注意：返回函数不要引用任何循环变量，否则后续会发生变化的变量

关键字lambda表示匿名函数

装饰器：

def log(func):
def wrapper(args,**kw):
print(‘call %s():’,func.name)
return func(args,**kw)
return warpper

@log
def now():
print(‘2015-3-25’)

注意：这里首先定义了一个装饰器，这个装饰器需要输入一个函数，但是我们在创建新的函数的时候，在函数定义的开头创建一个@装饰器名称，相当于：装饰器函数(刚创建的函数)，就会在刚创建的函数运行之后返回一个调用装饰器函数的结果
对于装饰器的理解，我百度了下

理解：程序本身就像内裤，内裤非常重要，执行一些非常重要的操作，但是不能一直穿，至少冬天穿的时候非常冷，于是就有了裤子，裤子把内裤包裹在里面的同时，提供的避寒的功能，这个裤子就是装饰器，在不影响函数整体运行的同时创建装饰器，很方便。

注意：创建一个装饰器名称是log,创建一个函数是now()
执行now()函数相当于执行 log(now)()函数
